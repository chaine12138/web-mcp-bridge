

## 一、已确认决策

| # | 决策点 | 结论 |
|---|---|---|
| 1 | 接入形态 | **本期仅纯 SDK**；未来可扩展 Chrome 扩展兜底 |

---

## 二、项目目标（单句）

> 交付一个 **Bridge（本地 MCP Server）** + **SDK（浏览器 npm 包）**，让已有 Web 应用用**最小代码**把自身业务能力注册为工具，供任意 MCP Agent 直接调用。

---

## 三、交付物（两个 package + 一份文档）

1. **`web-mcp-bridge`** — 本地进程
   - MCP Server over stdio（面向 Agent）
   - WebSocket Server `ws://127.0.0.1:<port>`（面向宿主 SDK）
   - CLI：`npx web-mcp-bridge --port 7321 --token xxx`
   - 动态聚合多会话工具、请求路由、超时、断线回收

2. **`web-mcp-sdk`** — 浏览器端 npm 包
   - `createMcpBridge({ appId, endpoint, token })`
   - `bridge.register({ name, description, inputSchema, handler })`
   - `bridge.connect() / disconnect()`
   - 自动重连 + 心跳 + 错误序列化
   - 类型：TS 一等公民，`inputSchema` 用 Zod

3. **接入文档** — `README` 一页纸 + 最小接入示例（不含 Demo 工程）

## 四、核心协议（Bridge ↔ SDK，JSON over WS）

```jsonc
// SDK → Bridge: 握手
{ "type": "hello", "appId": "lowcode-studio", "instanceId": "tab-abc", "token": "..." }

// SDK → Bridge: 声明/增量变更工具集
{ "type": "tools/register", "tools": [
   { "name": "updateComponent", "description": "...", "inputSchema": { /* JSON Schema */ } }
]}
{ "type": "tools/unregister", "names": ["..."] }

// Bridge → SDK: 调用工具
{ "type": "tool/call", "id": "req-1", "name": "updateComponent", "arguments": { ... } }

// SDK → Bridge: 回调
{ "type": "tool/result", "id": "req-1", "ok": true,  "data": { ... } }
{ "type": "tool/result", "id": "req-1", "ok": false, "error": { "code": "...", "message": "..." } }

// 双向心跳
{ "type": "ping" } / { "type": "pong" }
```

对 Agent 侧，工具名统一加命名空间：`<appId>.<toolName>`，多实例时 Bridge 自动做消歧（如让 Agent 先选 `instanceId`，或只对最新活动实例透传）。

## 五、范围边界

**In-scope（本期要做）**
- stdio MCP Server + 本地 WS Server
- SDK 的 `register / connect / disconnect / 自动重连`
- 多宿主会话并存 + 命名空间
- 基础鉴权：`token`（写死配置，通过环境变量或 CLI 参数传入）
- 请求超时、错误透传、日志

**Out-of-scope（本期不做）**
- Chrome 扩展形态
- 自动从 OpenAPI / TS 类型生成工具
- 宿主 Demo / 低代码示例
- 多用户权限 / 审计 / 可视化管理面板
- SSE / HTTP MCP 传输（只做 stdio）
- 生产级签名、CSP 兼容打包方案（先能用）

---

## 六、非功能约束

- **Node 版本**：≥ 18（ESM）
- **浏览器兼容**：Evergreen（Chrome/Edge/Firefox 最近两版）
- **SDK gzip 体积**：目标 < 15 KB（不含 Zod）
- **连接安全**：WS 仅绑 `127.0.0.1`，必须带 token
- **可观测**：Bridge 侧结构化日志（level、appId、toolName、latency）

## 八、验收场景（建议一条）

> 宿主应用（任意低代码平台）在页面初始化时注册 3 个工具：`updateComponent / deleteComponent / addComponent`。在 Qoder 里配置好 `web-mcp-bridge` 作为 MCP server，输入自然语言指令即可完成"新增按钮 → 改文案 → 删除某组件"，全程无需操作浏览器 UI。
